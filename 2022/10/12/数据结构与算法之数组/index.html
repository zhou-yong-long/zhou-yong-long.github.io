数据结构与算法之数组

数组的特点

  • 线性表数据结构（一般指数据只有前后两个方向，而非线性就是树结构）
  • 连续的内存空间和相同类型的数据（为什么是相同类型呢? 因为随机访问数组元素时需要计算偏移量：公式 = 内存块首地址 + 类型内存大小 * 下标 ）
  • 支持随机访问
  • 插入、删除较慢，查询较快
  • 存在访问越界问题

有趣的知识点: 为什么数组下标都是从 0 开始?

  • 第一点：假设下标为 x，计算该下标所对应的元素的内存地址公式为 address = base_address + x * type_size ，如果下标从 1 开始这个公式就会变成 address = base_address + (x - 1) * type_size，即下标从 1 开始会因为计算一次减法而增加时间消耗（可能消耗也不会很大）
  • 第二点：其次是由于一开始的 C 语言的数组下标是从 0 开始的，基于 C 语言开发的高级语言也不想因为从 1 开始而产生各种各样的 bug，其次是减少了学习成本。

知识映射：数组删除 => JVM 垃圾回收算法

假设场景如下：

依次从 数组：{a,b,c,d,e,f} 删除 {a,c,e}

由于每次从数组中删除元素都要移动后面的元素填上空缺，导致删除效率非常低。

可以效仿垃圾回收算法中的标记清除算法，记录需要删除的元素，直到数组没有空闲位置时才开始删除需要删除的元素。

垃圾回收算法

  • 标记删除：删除标记的对象，容易导致大量的内存碎片
  • 标记整理：标记生存的对象，并整理移动到最左边
  • 标记复制：将内存划分为两个区域——eden、survial，每次只使用
